Key Design Decisions

  • Domain-organized, not package-organized — users think in terms of "persistence" or "events", not "RCommon.EfCore"
  • Abstraction → Provider pattern — each section explains the abstraction first, then shows provider-specific setup
  • Progressive complexity — Getting Started → Core Concepts → Domain features → Architecture → Examples
  • Recipes section — real-world patterns combining multiple features together
  • Every page includes: concept overview, NuGet install, configuration, usage examples, provider comparison (where applicable)
